# SkillForge AI Academy 🚀

SkillForge AI Academy is a premium, production-ready full-stack educational platform inspired by industry leaders like Coursera and Analytics Vidhya. It is designed to provide a seamless learning experience for students while offering powerful management tools for instructors and admins.

## ✨ Core Features

### 🎓 For Students
- **Full-Stack Learning Dashboard**: Track progress, resume lessons, and browse purchased courses.
- **AI-Powered Learning**: Course recommendations and assistant features powered by Gemini 2.0.
- **Course Marketplace**: Explore, filter, and search through a diverse catalog of AI programs.
- **Secure Authentication**: Google OAuth and JWT-based email/password login via Firebase.
- **Interactive Video Player**: Modern video interface with progress tracking.

### 👨‍🏫 For Instructors
- **Course Management**: Create, edit, and publish AI modules and lessons.
- **Revenue Analytics**: Real-time sales tracking and enrollment statistics.
- **Student Engagement**: Monitor student progress across your curriculum.

### 🛡️ For Admins (Internal)
- **Role-Based Access Control**: Hardened Firestore security rules with ABAC (Attribute-Based Access Control).
- **System Moderation**: Global course and user management.

## 🛠️ Tech Stack

- **Frontend**: React 19, TypeScript, Vite, Tailwind CSS 4.0, Shadcn/UI, Framer Motion.
- **Backend**: Node.js, Express.js (Full-stack architecture).
- **Database & Auth**: Firebase Firestore & Firebase Authentication.
- **AI Engine**: Google Gemini 1.5/2.0 API.
- **Payments**: Stripe (Integrated API scaffolding).
- **State Management**: Zustand & React Query.

## 🚀 Getting Started (Development Mode)

Follow these steps to run the project locally or in a development container.

### 1. Prerequisites
- **Node.js**: v18+ or v20+ recommended.
- **npm**: v9+ or v10+.
- **Firebase Project**: Create a project in the [Firebase Console](https://console.firebase.google.com/).
- **Gemini API Key**: Obtain from [Google AI Studio](https://aistudio.google.com/).

### 2. Configuration
Create a `.env` file in the root directory and add the following:

```env
# Gemini AI
GEMINI_API_KEY="your_gemini_api_key"

# Stripe (Optional for payments)
STRIPE_SECRET_KEY="sk_test_..."
VITE_STRIPE_PUBLISHABLE_KEY="pk_test_..."

# Firebase Applet Config
# Usually generated by AI Studio, but you can manually add your firebase-applet-config.json
```

Ensure your `firebase-applet-config.json` is in the root directory.

### 3. Installation
```bash
# Install dependencies
npm install

# Initialize Shadcn (if starting from scratch)
npx shadcn@latest init
```

### 4. Running the Development Server
This project uses a full-stack architecture where Express serves the Vite middleware.

```bash
# Run the combined dev server (Frontend + Backend)
npm run dev
```
The app will be available at `http://localhost:3000`.

```text
├── server.ts             # Express entry point with Vite middleware
├── firestore.rules       # Hardened production-ready security rules
├── firebase-blueprint.json # Data schema and entity definitions
└── package.json          # Dependency and script management
```

## 🔒 Security Best Practices
- **Hardened Rules**: This project uses a "Master Gate" pattern for Firestore rules, ensuring that users can never access data they don't own.
- **Schema Validation**: Every write operation is validated through standalone logic helpers within Firestore rules.
- **Lazy Initialization**: SDKs (Stripe, Firebase) are initialized lazily to prevent startup crashes in environments missing secrets.
